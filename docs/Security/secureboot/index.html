🔐 Secure Boot（セキュアブート）とは？

Secure Bootは、マイコンが起動するときに実行されるプログラム（ファームウェア）が正しいかどうかをチェックする仕組みです。

🌟 なぜSecure Bootが必要？​

  • 悪意のある第三者が改ざんしたプログラムが動くのを防ぎたい
  • セキュリティ侵害によるシステム破壊や情報漏えいを防止したい

🛠 仕組み​

  1. マイコンの起動時に、まず**最初のプログラム（ブートローダ）**が動きます。
  2. ブートローダは、メインのファームウェアに対して「このプログラムは改ざんされていないか？」をチェックします。
  3. チェックは、プログラムに付けられた電子署名やハッシュ値を使って行います。
  4. 正しい署名があれば、プログラムを実行。
  5. 署名が無効・改ざんがあれば、起動を止めたり、エラーを報告します。

🔑 ポイント​

項目説明
電子署名作成者だけが持つ秘密鍵で生成し、改ざん検知に使う
ハッシュ値プログラムの内容を一意に表すデジタル指紋
ブートローダ最初に動くプログラム。署名検証などを担当
Root of Trust最初に信用できるコードが入った領域（ROMなど）

📌 メリット​

  • ファームウェアの改ざんを検出できる
  • 不正プログラムの起動を防げる
  • IoT機器の安全運用に必須の機能
